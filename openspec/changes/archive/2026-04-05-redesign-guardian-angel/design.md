## Context

三個特殊船員角色重新定位為各自獨立的情報/行動來源：

```
┌──────────────────────────────────────────────────────┐
│              船員特殊角色分工                           │
├──────────────┬──────────────┬────────────────────────┤
│  Engineer    │ Guardian Angel│  Doctor（新）           │
│  掃描存活者   │ 保護存活者    │  驗屍被淘汰者           │
│  ↓           │ ↓             │  ↓                    │
│  情報收集     │ 防禦          │  事後驗證               │
│  「他是Gnosia」│ 「擋住攻擊」  │ 「被投出的是/不是Gnosia」│
│  對象：活人   │ 對象：活人     │  對象：已淘汰者         │
│  時機：每夜   │ 時機：每夜     │  時機：有人被投票淘汰後  │
└──────────────┴──────────────┴────────────────────────┘
```

## Goals / Non-Goals

**Goals:**
- GA：無情報 + 保護 + 可公開身份
- Doctor：無保護 + 驗屍被投票者 + 隱藏身份為佳
- Engineer：100% 掃描存活者
- 三者互不重複，各有獨特價值

**Non-Goals:**
- 改變配置表人數
- 改變角色性格

## Decisions

### 1. 三角色對比

| 比較 | Engineer | Guardian Angel（新） | Doctor（新） |
|------|----------|-------------------|------------|
| 情報來源 | 掃描存活者 | 無 | 驗屍被淘汰者 |
| 夜間行動 | 掃描 1 人 | 保護 1 人 | 查看前日被投票者陣營 |
| 連續限制 | 不重複掃描 | **無限制** | 自動（只能查前日被投票者） |
| 身份策略 | 隱藏→關鍵時公開 | **鼓勵公開** | 隱藏為佳 |
| 對 Gnosia 的威脅 | 直接指認 | 擋住攻擊 | 事後驗證投票是否正確 |

### 2. Doctor 新能力：驗屍

**機制**：
- 夜間自動查看**當日被投票淘汰者**的陣營
- 結果：「是 Gnosia」或「不是 Gnosia」
- 若當日投票平票（無人被淘汰）→ Doctor 無法使用能力
- 第一天若有人被淘汰 → 第一夜可以驗屍

**策略價值**：
- 投票淘汰後，Doctor 可以驗證「我們投對了嗎？」
- 如果驗屍結果是「不是 Gnosia」→ 代表投錯人了，需要重新思考
- 如果驗屍結果是「是 Gnosia」→ 確認方向正確
- Doctor 選擇何時公開這個資訊是關鍵策略

**Prompt 設計**：
```
你是醫生。你的能力是查看前一日被投票淘汰者的陣營。
驗屍歷史：【列表】
昨日被投票淘汰者：【角色名】
驗屍結果：【角色名】【是/不是】Gnosia
（若昨日無人被淘汰：今夜無法使用能力）
```

**注意**：Doctor 不需要「選擇目標」——目標自動為當日被淘汰者。所以 Doctor 的夜間行動不需要 Launch agent，GM 自動解析即可。

### 3. Guardian Angel 設計（同前）

- 無情報、有保護力、可公開身份
- 可連續保護同一人
- 夜間 Launch agent 選擇保護目標

### 4. Engineer 恢復 100%

移除模糊掃描。三個角色各有明確情報來源，不需要用模糊來削弱。

### 5. Doctor 不需要 Agent 夜間行動

Doctor 的驗屍是自動的（目標固定為當日被淘汰者），GM 在夜間結果解析時直接計算結果，不需要 Launch Doctor agent。結果在下一日的討論 prompt 中傳給 Doctor。

## Risks / Trade-offs

- **Doctor 驗屍太弱？** → 但它提供的是「確認/否認」情報，在投錯人時極有價值
- **平票日 Doctor 無法使用** → 這是設計意圖，讓投票結果更重要
- **GA 成為唯一保護者** → GA 被消滅後無人保護，增加 Gnosia 策略空間
